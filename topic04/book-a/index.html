For example (do not use this example or a similar one for movies for your CA)

An XML file might be designed to store details on TV series.

You would expect to store data on the following:

show name, rating, year produced, creator, year aired, description, cast names, genres etc

You might start to structure the XML elements by using one Show as a template:

The show called Firefly has a rating of 9.2 and is classes as a parental guidance show. It is set five hundred years into the future, a renegade crew aboard a small space-craft try to survive as they travel the unknown parts of the galaxy, evading warring factions as well as authority agents out to get them. The show was created by Joss Whedon and season 1 was aired in 2002. The cast includes Nathan Fillion, Gina Torres, Alan Tudyk, and Summer Glau. The genres it is classified in are Sci-Fi, Adventure and Drama.

This is what you XML elements could look like:

<show id="001">
        <name>Firefly</name>
        <rating>9.2</rating>
        <age class="PG" />
        <desc>Five hundred years into the future, a renegade crew aboard a
            small space-craft try to
            survive as they travel the unknown parts of
            the galaxy, evading warring
            factions as well
            as authority agents out to
            get them.</desc>
        <creator>Joss Whedon</creator>
        <aired seasons="1">2002-2003</aired>
        <cast>
            <actor>Nathan Fillion </actor>
            <actor>Gina Torres </actor>
            <actor>Alan Tudyk </actor>
            <actor>Summer Glau </actor>
        </cast>
        <genres>
            <genre>Sci-Fi</genre>
            <genre>Adventure</genre>
            <genre>Drama</genre>
        </genres>
        <image src="images/firefly.jpg" />
    </show>

By the end of this lab you must have a working version of your XML elements. The XML file is expected to be submitted through Moodle on Wednesday 24th February.
